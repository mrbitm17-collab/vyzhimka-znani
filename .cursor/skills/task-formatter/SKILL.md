# Task Formatter — автоформатирование заданий

## Когда применять

При получении ЛЮБОГО задания от пользователя — перед началом работы.

## Алгоритм

### 1. Разбор задания

Из сообщения пользователя извлеки:
- **Что** нужно сделать (конкретные действия)
- **Зачем** (мотивация, если указана)
- **Какие файлы** затронуты
- **Какие ограничения** подразумеваются

### 2. Формирование структуры

Перед выполнением внутренне сформулируй задачу в формате:

```
Контекст: [Почему задача нужна, какой эффект ожидается]
Цель: [Конкретные измеримые действия — список]
Затрагиваемые файлы: [Какие файлы изменятся]
Ограничения: [Что НЕ менять, максимальный масштаб]
Проверка: [Как убедиться что всё работает]
```

### 3. Оценка масштаба

- **Малая задача** (1-3 файла, < 50 строк изменений): выполняй сразу
- **Средняя задача** (3-10 файлов): создай todo-список, выполняй по шагам
- **Большая задача** (10+ файлов или архитектурные решения): предложи план, дождись подтверждения

### 4. Проверка перед выполнением

- Задача точная и измеримая?
- Ограничения явно заданы или выведены из контекста?
- Нет конфликта с существующими правилами проекта?
- Масштаб контролируемый?

## Принципы (из дистиллятов)

- Мягкий директивный тон в коммуникации с пользователем
- Богатые промпты = богатые результаты: чем детальнее описание, тем лучше
- Контекст + мотивация для каждого правила и решения
- Ограничения масштаба всегда явные
- Исследование кодбейса перед изменениями

## Пример применения

Пользователь: "Добавь поиск на сайт"

Внутренняя структура:
```
Контекст: На сайте нет функции поиска, пользователи не могут искать видео/каналы
Цель: 
  1. Создать страницу /search с формой поиска
  2. Реализовать поиск по видео и каналам через Prisma
  3. Вывести результаты в карточках (существующий стиль)
Файлы: src/app/search/page.tsx (новый), возможно layout.tsx (уже есть форма)
Ограничения: Использовать существующий стиль карточек, не менять другие страницы
Проверка: Поиск находит видео по названию, каналы по имени
```
