# Skill Saver — сохранение задач как скилов

## Когда применять

1. После успешного завершения задачи, которая может повториться
2. Когда пользователь явно просит сохранить паттерн
3. При повторении задачи — обновить существующий скил улучшенной версией

## Алгоритм создания скила

### 1. Анализ завершённой задачи

- Какие шаги были выполнены?
- Какие файлы изменены?
- Какие решения приняты и почему?
- Были ли ошибки/откаты? Что помогло их избежать?

### 2. Создание скила

Расположение: `.cursor/skills/<skill-name>/SKILL.md`

Правила именования:
- Только строчные буквы, цифры, дефисы
- Максимум 64 символа
- Описательное имя: `sync-youtube-channel`, `redesign-page`, `add-prisma-model`

### 3. Структура SKILL.md

```markdown
---
name: <skill-name>
description: >-
  Что делает скил и когда применять. Включить ключевые слова
  для автоматического обнаружения.
---

# <Название>

## Шаги
1. Конкретный шаг с командой или действием
2. ...

## Файлы
- Какие файлы обычно затрагиваются

## Частые ошибки
- Ошибка и как избежать

## Пример
Конкретный пример применения
```

### 4. Обновление существующего скила

При повторении задачи:
1. Найди существующий скил в `.cursor/skills/`
2. Сравни с текущим выполнением
3. Обнови: добавь новые шаги, исправь устаревшие, улучши примеры
4. Пометь что это улучшенная версия

## Принципы

- Только специфичная для проекта информация, не общие знания
- Конкретные команды и файлы, не абстрактные советы
- Максимум 500 строк в SKILL.md
- Один скил = одна задача/паттерн
- Повторения → обновляй скил, не создавай дубликат
